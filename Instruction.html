Modern Minimalist Blog - Instructions

A clean, responsive Jekyll theme with dark mode support

Introduction

Welcome to the Modern Minimalist Blog theme! This is a clean, responsive Jekyll theme designed for bloggers who want a minimalist approach with modern aesthetics. The theme features:

  • Responsive design that works on all devices
  • Dark mode support with automatic OS preference detection
  • Easily customizable colors and typography
  • Clean typography with the Inter font family
  • Fast loading with minimal dependencies

Folder Structure

The project follows the standard Jekyll directory structure:

jekyll-blog-theme/
├── _config.yml
├── _layouts/
│   ├── default.html
│   ├── post.html
│   └── page.html
├── _includes/
│   ├── header.html
│   ├── footer.html
│   ├── nav.html
│   └── post-card.html
├── _posts/
│   └── 2025-07-24-welcome.md
├── assets/
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── main.js
├── index.md
├── demo_index.html
└── Instruction.html

Key Files Explanation

  • _config.yml: Main configuration file for site settings
  • _layouts/: Template files that define the structure of pages
  • _includes/: Reusable components like header, footer, navigation
  • _posts/: Blog posts in Markdown format
  • assets/: CSS and JavaScript files
  • index.md: Homepage content
  • demo_index.html: Standalone HTML preview of the homepage
  • Instruction.html: This documentation file

Customizing the Theme

Changing Colors

To change the color scheme, modify these parameters in _config.yml:

theme_color: "#3498db"  # Primary accent color

For more advanced color customization, edit the CSS variables in assets/css/style.css:

:root {
  --primary-color: #3498db;
  --text-color: #333;
  --bg-color: #fff;
  --border-color: #eee;
  --link-color: #3498db;
  --hover-color: #2980b9;
}

Changing Fonts

Update the font_family parameter in _config.yml:

font_family: "'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, 'Open Sans', 'Helvetica Neue', sans-serif"

Adding Content

Creating New Posts

To add a new blog post:

  1. Create a new file in the _posts directory
  2. Name it using the format: YYYY-MM-DD-title.md
  3. Add front matter at the top of the file:
---
layout: post
title: "Your Post Title"
date: 2025-07-25 12:00:00 +0000
categories: jekyll update
author: Your Name
---

Then write your content in Markdown below the front matter.

Adding Pages

To create a new static page:

  1. Create a new Markdown file in the root directory
  2. Add front matter specifying the layout:
---
layout: page
title: "About"
---

Using demo_index.html

The demo_index.html file is a standalone HTML preview of the homepage. It's useful for:

  • Previewing the design without running Jekyll
  • Sharing a static version of the layout
  • Testing design changes quickly

To use it:

  1. Open demo_index.html directly in a web browser
  2. All styles and functionality are contained in the file
  3. No Jekyll or server required
